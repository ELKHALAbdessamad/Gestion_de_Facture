\chapter{Spécification des Besoins}

Ce chapitre est le « contrat fonctionnel » entre les parties prenantes et l'équipe technique. Il détaille ce qui doit être fait dans le périmètre défini au Chapitre 3.

\subsection{Profils utilisateurs}

\begin{description}
    \item[\textbf{USER :}] Création/modification factures, gestion clients, PDF, suivi paiements. Pas d'accès paramètres système.
    \item[\textbf{ADMIN :}] Hérite de USER + validation, suppression, gestion articles/catégories/paramètres, dashboard global.
\end{description}

\subsection{Limites nécessitant un backend}

\rowcolors{2}{lightgray}{white}
\begin{table}[H]
\centering
\caption{Limites actuelles et prérequis backend pour la production}
\label{tab:limites_backend}
\begin{tabularx}{\textwidth}{|l|X|}
\hline
\rowcolor{emsiblue}\textcolor{white}{\textbf{Point}} & \textcolor{white}{\textbf{Note}} \\
\hline

JWT réel & localStorage démo — Firebase Auth activable pour JWT sécurisé \\
Archivage annuel & Export ZIP disponible — stockage cloud + rétention serveur requis pour légalité 10 ans \\
Multi-devise/société & Interface disponible — architecture comptable complète prévue en V2 \\
\hline
\end{tabularx}
\end{table}
